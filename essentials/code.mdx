Mintlify provides powerful code block features with syntax highlighting, file names, line highlighting, and multi-language examples.

## Inline Code

Wrap code in single backticks to display it inline: `const greeting = "Hello";`

```md
Wrap code in backticks: `const greeting = "Hello";`
```

## Basic Code Blocks

Create code blocks using triple backticks with a language identifier:

```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("World");
```

````md
```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}
```
````

## Line Highlighting

Highlight specific lines to draw attention to important code:

```javascript {3-5}
function calculateTotal(items) {
  let total = 0;
  for (const item of items) {
    total += item.price;
  }
  return total;
}
```

````md
```javascript {3-5}
function calculateTotal(items) {
  let total = 0;
  for (const item of items) {
    total += item.price;
  }
  return total;
}
```
````

Highlight individual lines or ranges:

- Single line: `{3}`
- Multiple lines: `{1,3,5}`
- Range: `{3-8}`
- Combined: `{1,3-5,8}`

## Diff Syntax

Show code changes using `+` and `-` prefixes:

```diff
function greet(name) {
-  console.log("Hello!");
+  console.log(`Hello, ${name}!`);
}
```

````md
```diff
function greet(name) {
-  console.log("Hello!");
+  console.log(`Hello, ${name}!`);
}
```
````

## Best Practices

<Note>
**Always specify a language** for syntax highlighting. This improves readability and helps users understand the code context.
</Note>

<Tip>
**Use CodeGroup for API examples** showing the same operation in different languages. This helps developers working in their preferred language.
</Tip>

<Warning>
**Keep code blocks concise**. For lengthy code, consider linking to a GitHub repository or using expandable sections.
</Warning>

## Terminal Output

For terminal commands and output, use `bash` or `shell`:

```bash
npm install mintlify
```

For multi-step terminal workflows:

```bash

npm install

# Start development server
npm run dev

# Build for production
npm run build
```
