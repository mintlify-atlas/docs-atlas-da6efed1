Every Mintlify site needs a `docs.json` file with the core configuration settings. This file controls your site's appearance, navigation structure, branding, and functionality.

## Basic Configuration

<ParamField path="name" type="string" required>
  Name of your project. Used for the global title.

  ```json
  "name": "Mint Starter Kit"
  ```
</ParamField>

<ParamField path="theme" type="string">
  The theme for your documentation. Common value is `"mint"`.

  ```json
  "theme": "mint"
  ```
</ParamField>

## Branding

### Logo

<ParamField path="logo" type="string | object">
  Path to logo image or object with paths to light and dark mode logo images.

  <Accordion title="Logo Configuration">
    <ParamField path="light" type="string">
      Path to the logo in light mode
    </ParamField>
    <ParamField path="dark" type="string">
      Path to the logo in dark mode
    </ParamField>
    <ParamField path="href" type="string" default="/">
      Where clicking on the logo links you to
    </ParamField>
  </Accordion>
</ParamField>

```json Example
"logo": {
  "light": "/logo/light.svg",
  "dark": "/logo/dark.svg"
}
```

### Favicon

<ParamField path="favicon" type="string">
  Path to the favicon image displayed in browser tabs.

  ```json
  "favicon": "/favicon.svg"
  ```
</ParamField>

### Colors

<ParamField path="colors" type="object" required>
  Hex color codes for your global theme. These colors are used throughout your documentation for accents, highlights, and interactive elements.
</ParamField>

<CodeGroup>
```json Basic Colors
"colors": {
  "primary": "#16A34A",
  "light": "#07C983",
  "dark": "#15803D"
}
```

```json With Background
"colors": {
  "primary": "#16A34A",
  "light": "#07C983",
  "dark": "#15803D",
  "background": {
    "light": "#ffffff",
    "dark": "#0a0a0a"
  }
}
```
</CodeGroup>

<Accordion title="Color Properties">
  <ParamField path="primary" type="string" required>
    The primary color used for highlighted content, section headers, and accents in light mode
  </ParamField>
  <ParamField path="light" type="string">
    The primary color for dark mode
  </ParamField>
  <ParamField path="dark" type="string">
    The primary color for important buttons
  </ParamField>
  <ParamField path="background" type="object">
    Custom background colors for light and dark mode
  </ParamField>
</Accordion>

## Navigation Bar

### Navbar Links

<ParamField path="navbar" type="object">
  Configure the navigation bar at the top of your documentation.
</ParamField>

```json
"navbar": {
  "links": [
    {
      "label": "Support",
      "href": "mailto:hi@mintlify.com"
    }
  ],
  "primary": {
    "type": "button",
    "label": "Dashboard",
    "href": "https://dashboard.mintlify.com"
  }
}
```

<Accordion title="Navbar Configuration">
  <ParamField path="links" type="array">
    Array of link objects with `label` and `href` properties
  </ParamField>
  <ParamField path="primary" type="object">
    Primary call-to-action button configuration with `type`, `label`, and `href`
  </ParamField>
</Accordion>

<Tip>
  The primary button appears prominently in your navbar and can be styled as a button or link to external resources like your dashboard or app.
</Tip>

## Footer

<ParamField path="footer" type="object">
  Configure footer settings including social media links.
</ParamField>

```json
"footer": {
  "socials": {
    "x": "https://x.com/mintlify",
    "github": "https://github.com/mintlify",
    "linkedin": "https://linkedin.com/company/mintlify"
  }
}
```

<Note>
  Supported social platforms include: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, and `hacker-news`.
</Note>

## API Configuration

<ParamField path="api" type="object">
  Configuration for API settings and playground behavior.
</ParamField>

<Accordion title="API Settings">
  <ParamField path="baseUrl" type="string | string[]">
    The base URL for all API endpoints. Can be an array to enable multiple base URL options
  </ParamField>
  <ParamField path="auth" type="object">
    Authentication configuration with `method` (bearer, basic, or key), `name`, and optional `inputPrefix`
  </ParamField>
  <ParamField path="playground" type="object">
    Playground settings with `mode` option: `"show"`, `"simple"`, or `"hide"`
  </ParamField>
  <ParamField path="maintainOrder" type="boolean">
    Ensures key ordering in OpenAPI pages matches the OpenAPI file definition
  </ParamField>
</Accordion>

<ParamField path="openapi" type="string | string[]">
  URL(s) or relative path(s) pointing to your OpenAPI specification file.
</ParamField>

<CodeGroup>
```json Absolute URL
"openapi": "https://example.com/openapi.json"
```

```json Relative Path
"openapi": "/openapi.json"
```

```json Multiple Files
"openapi": [
  "https://example.com/openapi1.json",
  "/openapi2.json",
  "/openapi3.json"
]
```
</CodeGroup>

## Advanced Features

### Mode Toggle

<ParamField path="modeToggle" type="object">
  Customize the dark mode toggle behavior.
</ParamField>

<CodeGroup>
```json Dark Mode Only
"modeToggle": {
  "default": "dark",
  "isHidden": true
}
```

```json Light Mode Only
"modeToggle": {
  "default": "light",
  "isHidden": true
}
```

```json Default to System
"modeToggle": {
  "default": "system"
}
```
</CodeGroup>

### Feedback Buttons

<ParamField path="feedback" type="object">
  Enable feedback buttons to allow users to suggest edits or raise issues.
</ParamField>

```json
"feedback": {
  "suggestEdit": true,
  "raiseIssue": true
}
```

### Background Image

<ParamField path="backgroundImage" type="string">
  A background image to be displayed behind every page.

  ```json
  "backgroundImage": "/background.png"
  ```
</ParamField>

### Versions

<ParamField path="versions" type="string[]">
  Array of version names to show different versions of docs with a dropdown in the navigation bar.

  ```json
  "versions": ["v2.0", "v1.0"]
  ```
</ParamField>

## Contextual Features

<ParamField path="contextual" type="object">
  Configure contextual menu options for code blocks and content.
</ParamField>

```json
"contextual": {
  "options": [
    "copy",
    "view",
    "chatgpt",
    "claude",
    "perplexity",
    "mcp",
    "cursor",
    "vscode"
  ]
}
```

<Tip>
  Contextual options provide quick actions for users to interact with your documentation, like copying code or opening in their preferred AI assistant.
</Tip>

## Complete Example

Here's a complete `docs.json` configuration from the Mintlify Starter Kit:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Mint Starter Kit",
  "colors": {
    "primary": "#16A34A",
    "light": "#07C983",
    "dark": "#15803D"
  },
  "favicon": "/favicon.svg",
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg"
  },
  "navbar": {
    "links": [
      {
        "label": "Support",
        "href": "mailto:hi@mintlify.com"
      }
    ],
    "primary": {
      "type": "button",
      "label": "Dashboard",
      "href": "https://dashboard.mintlify.com"
    }
  },
  "footer": {
    "socials": {
      "x": "https://x.com/mintlify",
      "github": "https://github.com/mintlify",
      "linkedin": "https://linkedin.com/company/mintlify"
    }
  }
}
```

<Note>
  For complete navigation configuration, see the [Navigation](/essentials/navigation) guide.
</Note>
