<Info>
  **Prerequisites**:
  - Node.js version 19 or higher
  - A docs repository with a `docs.json` file
</Info>

Follow these steps to install and run Mintlify on your operating system.

<Steps>
<Step title="Install the Mintlify CLI">

Install the Mintlify CLI globally using your preferred package manager:

<CodeGroup>
```bash npm
npm i -g mint
```

```bash yarn
yarn global add mint
```

```bash pnpm
pnpm add -g mint
```
</CodeGroup>

</Step>

<Step title="Preview locally">

Navigate to your docs directory where your `docs.json` file is located, and run the following command:

```bash
mint dev
```

A local preview of your documentation will be available at `http://localhost:3000`.

<Tip>The preview server supports hot reloading. Your changes will appear automatically as you edit and save files.</Tip>

</Step>
</Steps>

## File structure

Your Mintlify documentation project includes:

- **docs.json** - Configuration file for your site settings, navigation, and theme
- **MDX files** - Your documentation pages written in Markdown with JSX components
- **images/** - Directory for images and assets referenced in your docs
- **logo/** - Your brand logos for light and dark modes

<Note>Make sure to run `mint dev` in the folder containing your `docs.json` file. If a page loads as a 404, verify you're in the correct directory.</Note>

## Custom ports

By default, Mintlify uses port 3000. You can customize the port Mintlify runs on by using the `--port` flag:

```bash
mint dev --port 3333
```

If you attempt to run Mintlify on a port that's already in use, it will automatically use the next available port:

```
Port 3000 is already in use. Trying 3001 instead.
```

## CLI commands

### Update the CLI

Keep your CLI up to date to ensure your local preview matches the production version:

```bash
npm mint update
```

Alternatively, run `mint update` to update to the latest version.

### Validate links

The CLI can help you identify broken links in your documentation:

```bash
mint broken-links
```

This command scans all pages and reports any broken internal or external links.

## Code formatting

We suggest using extensions on your IDE to recognize and format MDX. If you're a VSCode user, consider:

- [MDX VSCode extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) for syntax highlighting
- [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) for code formatting

These tools make it easier to write well-formatted documentation with proper syntax highlighting.

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>
    This may be due to an outdated version of Node.js. Try the following:
    
    1. Remove the currently-installed version of the CLI:
    ```bash
    npm remove -g mint
    ```
    
    2. Upgrade to Node v19 or higher using [nvm](https://github.com/nvm-sh/nvm) or download from [nodejs.org](https://nodejs.org/).
    
    3. Reinstall the CLI:
    ```bash
    npm i -g mint
    ```
  </Accordion>

  <Accordion title="Dev environment isn't running">
    Run the following command to ensure you have the most recent version of the CLI:
    
    ```bash
    mint update
    ```
    
    Then try running `mint dev` again.
  </Accordion>

  <Accordion title="Encountering an unknown error">
    Go to the root of your device and delete the `~/.mintlify` folder:
    
    ```bash
    rm -rf ~/.mintlify
    ```
    
    Then run `mint dev` again. This clears the local cache and resolves most persistent issues.
  </Accordion>
  
  <Accordion title="Page loads as a 404">
    Make sure you are running `mint dev` in a folder with a valid `docs.json` file. The CLI needs this configuration file to properly serve your documentation.
  </Accordion>
</AccordionGroup>
